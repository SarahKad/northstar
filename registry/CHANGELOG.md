## [Unreleased]

### Added

- **`index.html`**, offline, self-contained Getting Started (no sidebar); generated by `npm run start-here:generate`
- **Getting Started**, `/docs/getting-started` with Terminal, Cursor, and Claude Code tabs; Cursor tab uses `@ns-get-started` (bootstrap) then `@ns-vibe-start` (app shell or building)
- **`ns-get-started` skill**, `.cursor/skills/ns-get-started/` bootstrap workflow; hands off to `ns-vibe-start`
- **`npm run dev:install`**, registry on port 3002 for background component installs (Cursor/agent); keeps 3000 for the user's app
- **Cursor browser guidance**, skills and docs warn not to preview the registry during setup; design system docs ≠ user's app

### Changed

- **Getting Started (Cursor)**, Step 0: save design system kit locally, open **project folder** in Cursor (not registry); Step 1: Bootstrap + Shell with `REGISTRY_DIR` + `MY_APP_DIR` paths
- **`ns-get-started` skill**, continues to `@ns-vibe-start` when both skills are invoked; stops after bootstrap when user asks for bootstrap-only
- **Table registry item**, ships North Star `table.tsx` (muted header, stripes, `TableHeadSort`) instead of stock shadcn table
- **Stepper**, labels align under step circles; full-width layout
- Setup skills instruct agents not to open Simple Browser for the registry; stop install server before starting the app on port 3000
- **App shell template**, `templates/app-shell/` with demo `/login`, `/dashboard` showcase, and `npm run scaffold:shell`
- **Getting Started**, merged former App Shell Set Up page (install, ThemeProvider, verify steps) into Terminal/Cursor/Claude tabs; `/docs/app-shell` redirects to Getting Started
- **`useDocumentThemeId`** hook, portable chart theming without registry-only imports

### Fixed

- Chart previews, taller playground area and min-heights on chart renderers
- Chart registry items, include `card`, `select`, and `use-document-theme` dependencies where needed

## [0.1.0], 2026-05-21

### Added

#### Foundations
- Color system using OKLCH color space with three-tier token architecture (Primitive → Semantic → Component)
- Brand red `#DA291C` as `--brand` / `--destructive` / `--ring` across light and dark themes
- Chart palette anchored to `--muted-foreground` (`#7C6D67`) with 5-step lightness scale
- Typography tokens: Atkinson Hyperlegible Next (body), Instrument Serif (display), Geist Mono (code)
- Border radius token system via `--radius` base with `calc()` multipliers
- Spacing scale documentation (4px base unit, steps 1–24)
- Focus ring system via `outline-ring/50` in `@layer base`

#### Components, Inputs & Actions
- **Button**, 6 variants (default, secondary, outline, ghost, destructive, link), 5 sizes
- **Button Group**, connected segmented control with active state
- **Checkbox**, Base UI primitive, animated check, focus ring
- **Input**, type-aware with icon slot (text, email, password, number, search, url)
- **Radio Group**, Base UI primitive, animated dot, hover/press states
- **Select**, Base UI primitive, full keyboard navigation

#### Components, Display
- **Badge**, 4 variants (default, secondary, outline, destructive)
- **Card**, compound (Header, Title, Description, Content, Footer), optional image
- **Table**, compound with caption, footer, sortable-ready structure

#### Components, Layout
- **Separator**, horizontal/vertical rule

#### Components, Feedback
- **Alert**, 5 variants (default, info, success, warning, destructive), compound (Icon, Content, Title, Description, Dismiss)
- **Skeleton**, 3 presets (text, avatar, card)

#### Components, Navigation
- **Pagination**, prev/next arrows, numbered pages, active state, ellipsis
- **Side Navigation**, section-based with active link state
- **Tabs**, 2 variants (default, line), Base UI primitive

#### Components, Charts & Graphs
- **Area Chart** (interactive), stacked area with time-range selector, hatch pattern on secondary series
- **Bar Chart**, single series
- **Bar Chart Stacked**, two-series with SVG hatch pattern on secondary
- **Line Chart**, smooth curve with dot markers

#### Registry & Tooling
- `public/registry.json`, machine-readable component manifest for AI tools and MCP servers
- `npm run registry:generate`, script to regenerate manifest from source
- Figma Code Connect wired for Card component (test integration)
- `src/lib/nav-config.ts`, single source of truth for sidebar, breadcrumbs, prev/next, and ⌘K search
- Global search (⌘K) command palette via cmdk
- Right-side table of contents with scroll-aware active heading
- Breadcrumb navigation on all doc pages
- Prev/Next page navigation

#### Documentation
- Getting Started page
- Foundations section: Color Theming, Typography, Borders & Focus, Spacing, Iconography
- Component playground with live preview, props configurator, and code output
